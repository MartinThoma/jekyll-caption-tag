    <h1 id=“label-jekyll-caption-tag”>jekyll-caption-tag<span><a href=“#label-jekyll-caption-tag”>&para;</a> <a href=“#documentation”>&uarr;</a></span></h1>

    <p>A Liquid tag for Jekyll sites that allows easy creation of captioned images like they are in WordPress.</p>

    <h1 id=“label-How+To+Install”>How To Install<span><a href=“#label-How+To+Install”>&para;</a> <a href=“#documentation”>&uarr;</a></span></h1> <ol><li> <p>Install all required gems: dimensions, rmagick (sudo gem install dimensions rmagick on Debian-based machines)</p> </li><li> <p>Copy caption_tag.rb into &lt;your-jekyll-site&gt;/_plugins.</p> </li><li> <p>Add the styles from style.css to your stylesheets.</p> </li></ol>

    <h1 id=“label-How+To+use”>How To use<span><a href=“#label-How+To+use”>&para;</a> <a href=“#documentation”>&uarr;</a></span></h1>

    <p>Place a caption tag in your content file, e.g.: {% caption align=&quot;aligncenter&quot; width=&quot;512&quot; height=&quot;233&quot; alt=&quot;The order of points is important for the definition of a polygon&quot; text=&quot;[A, B, C, D, E, F, G] != [A, B, C, D, F, E, G]&quot; url=&quot;../images/2013/11/polygon-order.png&quot; %} </p>

    <p>| Attribute | Details | | ————- |————————————————————————— | | align | The content of this will be added to the class of the surrounding &lt;div&gt;. | | width | The width of the image. | | height | The width of the image. | | alt | The alternative attribute for images. | | caption | The text below the image. | | url | The source (src) of the image. |</p>

    <h1 id=“label-Examples”>Examples<span><a href=“#label-Examples”>&para;</a> <a href=“#documentation”>&uarr;</a></span></h1> <ul><li> <p><a href=“postgithub.com/MartinThoma/MartinThoma.github.io/blob/source/_posts/2013-11-18-check-point-inside-polygon.md”>post>, <a href=“rendered”>martin-thoma.com/check-point-inside-polygon/“>rendered post</a></p> </li></ul>

    <h1 id=“label-Changelog”>Changelog<span><a href=“#label-Changelog”>&para;</a> <a href=“#documentation”>&uarr;</a></span></h1>

    <p>Version 1.4, 2014-03-27: - Fixed issue (if two files had the same name, but were in different folder the captions were overwritten) Version 1.3, 2014-03-24: - Captions can resizes automatically to a given maximum size while preserving the aspect ratio.
